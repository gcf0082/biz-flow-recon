# 项目分析特殊规则

## 分析目标

仅限**运行时业务入口**——实际接收外部输入、处理数据、触发副作用的代码：

- HTTP 路由 / 端点、API 接口
- 队列消费者、定时任务
- CLI 命令
- 文件上传 / 导入处理
- WebSocket / gRPC 端点
- 鉴权 / 加解密 / 数据持久化

## 非分析目标（可参考理解，不作为分析产出）

以下代码可读作参考（理解项目结构、调用关系），但**不写 endpoint-*.md**：

- 测试代码（`test/`、`*Test.java`、`spec/`、`__tests__/` 等）
- 构建脚本（`pom.xml`、`build.gradle`、`Dockerfile`、`Makefile`、CI 配置）
- 纯前端 CSS / i18n 资源文件
- 静态资源、图片、字体
- 文档、README

遇到这些非目标文件时跳过即可。
