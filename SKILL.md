# biz-flow-recon

像一位熟悉系统的工程师，把代码讲给安全测试人员听。先一段连贯的话讲清"它在做什么"，再按子功能分组，每个接口下用**自然的段落**讲它怎么跑——文件 I/O、外部命令、第三方交互、加解密自然地织进句子里。判断权留给读者。

## 原则

- **描述事实，不评判**。禁用"风险/漏洞/不安全/缺乏校验/可能被攻击/建议加固"等词。
- **每条业务流用 1-2 段自然语言讲，不用 bullet**。事实嵌在句子里，简单接口可一句话讲完。
- **不存在的事实不提**，让产物像讲解、不像填表。
- **优先跨信任边界的动作**（用户输入入口、I/O、外呼、加解密），而不是内部纯计算。

## 取舍

**保留**：用户输入入口、文件读写、外部进程调用、网络外呼、加解密/签名、鉴权与会话、数据持久化。

**砍掉**：纯 CSS/样式、i18n 文案、内部纯计算函数、单元测试、构建脚本、健康检查、静态资源、Spring Actuator 之类纯运维端点。

## 方法

### 步骤 0：加载工作目录（每次都做）

1. 在当前工作目录下找 `biz-flow-recon/`：
   - 如果存在 `biz-flow-recon/knowledge/`，**把里面所有 `.md` 都读一遍**当先验知识，分析时尊重其中的术语、约定、模块说明。
   - 模板路径按这个顺序解析：`<cwd>/biz-flow-recon/templates/{name}.md`（项目覆盖）→ `<skill-dir>/templates/{name}.md`（默认）。
2. 如果当前目录下还没有 `biz-flow-recon/output/`，分析完后再创建——别预先建。

### 步骤 1：定粒度（默认 B）

- **默认走粒度 B**：按子功能分组（登录、下单、文件上传……），每组下列出相关接口。模板用 `templates/default.md`。
- **粒度 A（整项目）**：仅当用户明确说"整个项目""全量梳理""系统全貌"等总览意图时使用。模板 `templates/overview.md`。
- **粒度 C（单接口）**：仅当用户给了具体 URL 或类#方法名时使用。模板 `templates/endpoint.md`。

### 步骤 2-4：分析

2. **盘入口**：在选定范围内找 `@RestController`/`@Controller`/`@RequestMapping`、`@Scheduled`、MQ 监听器、CLI `main`、前端 router。
3. **跟链路**：每个入口跟到底——Service、Mapper/Repository、配置文件（`application.yml`）、文件 I/O、外部进程、网络调用、加密点。
4. **写段落**：照模板的骨架，把抓到的事实**编织成一两段散文**讲清楚，自然提到 `(类#方法，文件:行号)` 作为可定位锚点。

### 步骤 5：写产物

按下面的文件名约定写到 `<cwd>/biz-flow-recon/output/`（目录不存在就建）：

| 粒度 | 文件名 |
|---|---|
| A 整项目 | `overview.md` |
| B 子功能（默认全量） | `features.md` |
| B 子功能（限定范围，如"支付下单流程"） | `features-{slug}.md`（如 `features-pay-order.md`） |
| C 单接口 | `endpoint-{METHOD}-{path-slug}.md`（如 `endpoint-POST-api-login.md`） |

slug 规则：小写、连字符；中文/特殊字符转拼音或英文同义词。**默认覆盖同名文件**，便于 git diff。

写完告诉用户产物路径。

## 段落要包含什么

每条接口的描述段必须自然交代：

- 是哪段代码的哪个入口（含 `文件:行号`）
- 谁在触发
- 当**实际发生**时把以下信息织进描述：读写了哪些文件（路径+格式）、跑了哪些命令（含目的）、调了哪些第三方（协议、字段、鉴权）、做了哪些加解密（算法+对象+为什么）

不存在的不写。读者读完一段，应能跳到代码并知道这段流程在做什么。

## 工作目录约定

详见 `docs/workspace-layout.md`。简言之：被分析项目根目录下放 `biz-flow-recon/{knowledge,templates,output}/`，前两个用户填，最后一个 skill 写。

## 输出格式

格式由 `templates/` 下的模板定义，用户可在项目里覆盖。每次输出前**先读相应模板**再照样写，**不要凭记忆写**——这样用户改了模板能立即生效。
